WEB Augmented Reality

1. What Is Web AR?

Web AR is Augmented Reality that runs directly in a web browser without installing a mobile app.

It uses:

The phone camera

Motion sensors

The web browser

JavaScript APIs

 Key idea:

Web AR brings AR experiences to anyone with a link.

2. How Web AR Works (Simple Flow)

User opens a website

Browser asks permission for camera & sensors

WebXR connects to device tracking

The system detects surfaces

Virtual objects are placed in real space

Everything happens in real time.

4. Key Web AR Concepts (Must Know)

Reference Space

Defines how the virtual world aligns with the real world.

Examples:

Viewer space → relative to camera

Local space → room-based

Floor space → gravity aligned

Wrong reference space = floating or sliding objects.

 Hit Testing

Hit testing finds real-world surfaces.

A virtual ray is sent from the camera

It checks where it hits a surface

Used for placing objects on floors or tables

 Hit testing depends on:

Lighting

Surface texture

Camera quality

 Anchors

Anchors keep objects fixed in space.

Weak anchors → drift

Strong anchors → stable AR

 Web anchors are less stable than native AR.

 Tracking

Tracking updates:

Position

Rotation

Movement

Tracking is continuous and never perfect.

 Scale & Units

Web AR uses meters, not pixels

Object size must match real-world scale

 Wrong scale breaks realism instantly.

5. Web AR Limitations (Important)
Issue	                 Reason
Drift	                Sensor noise
Occlusion errors	No depth data
Lighting sensitivity	Camera-based tracking
Device differences	Not all phones support the same features

 Web AR chooses accessibility over precision.

8. Web AR vs Native AR

Runs in browser	Requires app
Easy access	Better performance
Faster to share	More precise tracking
Limited sensors	Full sensor access
